# Правила разработки

## Качество кода

- Пиши код как **сеньор-разработчик** — чистый, читаемый, поддерживаемый.
- Следуй принципам **SOLID**, **DRY**, **KISS**, **YAGNI**.
- Применяй лучшие практики архитектуры: слоистая архитектура, разделение ответственности, dependency injection.
- Не допускай "quick & dirty" решений. Каждое решение должно быть **production-ready**.
- Покрывай edge-cases, обрабатывай ошибки корректно.
- Используй типизацию везде, где это возможно (TypeScript strict mode, Python type hints).

## Vue 3 & Quasar Framework (Frontend)

- **Основной фреймворк:** Весь фронтенд проекта теперь разрабатывается на **Quasar Framework**. Желательно максимально использовать готовые компоненты Quasar.
- **Декомпозиция:** Максимально декомпозируй UI-компоненты и сторы. Запрещается писать всю логику и верстку в одном большом компоненте — необходимо создавать небольшие и сфокусированные компоненты.
- **Переиспользование:** Если в интерфейсе есть повторяющиеся элементы, обязательно выноси их в отдельный компонент и переиспользуй.
- **Структура файлов:** Для создания и размещения компонентов в директории проекта используй лучшие практики (папки `components`, `pages`, `layouts`, `composables`).
- **Архитектура Stores (Feature-Based):** Для каждой фичи (feature) в директории `stores/` должна создаваться отдельная папка. Внутри этой папки необходимо хранить всё, что относится к фиче: сам стор, TS-модели/интерфейсы и сервисы для работы с API этой фичи.
  - **Pinia Stores:** Обязательно использовать **классический Options Store** синтаксис (с объектом `state`, `actions`, `getters`). Использование Setup Store (через функцию-коллбэк `defineStore('name', () => {...})`) строго запрещено.
- **Стилизация:**
  - Писать стили **исключительно на SASS** (`<style lang="sass" scoped>`). Использование CSS или SCSS запрещено.
  - Использовать **только** переменные из файла `quasar/arbitration-art-q/src/css/quasar.variables.sass`.
  - Использовать набор **не более чем из 4 размеров шрифтов**, определенных в `quasar/arbitration-art-q/src/css/quasar.variables.sass`.
  - Использовать и дописывать общий **style guide** проекта (общие утилиты и классы для карточек, теней, кнопок, шрифтов и т.д.) вместо хардкода стилей.
  - Все кнопки (`q-btn`) должны использовать свойство `no-caps`, чтобы избежать автоматического перевода текста в верхний регистр.

## Язык общения

- Планирование, обсуждение архитектуры и ответы — **на русском языке**.
- **Комментарии в коде**: Исключительно на **английском языке** (STRICT RULE). Использовать русский язык в исходном коде строго запрещено.
- **Документация**: На английском языке.

## Выполнение команд

- **Не выполняй команды в терминале самостоятельно.**
- Вместо этого — выписывай команды, которые нужно выполнить, и пользователь выполнит их сам.
- Формат: блок кода с командой и краткое пояснение, что она делает.
